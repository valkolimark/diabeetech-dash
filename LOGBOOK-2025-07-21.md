## How the App Handles Bridge Protocol

### Initialization Sequence:
1. **Server Boot** (`bootevent-multitenant.js:308-316`)
   - Creates bridge manager instance
   - Calls `initializeAll()` after boot completes

2. **Bridge Manager** (`bridge-manager.js`)
   - Lists all active tenants from master database
   - For each tenant:
     - Connects to tenant database
     - Looks for settings document
     - Checks if bridge is enabled
     - Starts bridge if enabled

3. **Bridge Protocol** (`bridge-multitenant.js`)
   - Polls Dexcom API every 2.5 minutes (150000ms)
   - Fetches glucose readings
   - Stores in tenant's entries collection
   - Emits 'data-received' event for WebSocket updates
